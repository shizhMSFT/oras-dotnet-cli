# What's New in v0.4.0

**ORAS .NET CLI v0.4.0** marks a major milestone in the project's journey вЂ” **all 21 commands are now fully implemented with real OCI registry operations**. Previously, only 3 commands (login, logout, version) had working implementations. This release transforms the CLI from a prototype into a production-ready tool with complete parity to the Go ORAS CLI.

All 18 previously-stub commands now have real implementations calling the official **OrasProject.Oras v0.5.0** library:

### **Artifact Operations**
- **`push`** вЂ” Push files to a registry as an OCI artifact using `Packer.PackManifestAsync` and `CopyAsync`
- **`pull`** вЂ” Pull artifact files from a registry (fetches manifest, downloads layers to disk)
- **`attach`** вЂ” Attach files to an existing artifact as a referrer (signatures, SBOMs, attestations)
- **`copy`** вЂ” Copy an artifact between registries using `CopyAsync` with progress tracking
- **`discover`** вЂ” Discover referrers of a manifest (find attached signatures, SBOMs, etc.)
- **`resolve`** вЂ” Resolve a tag to a digest (get the immutable SHA256 reference)
- **`tag`** вЂ” Tag a manifest in the registry (add additional tags to existing artifacts)

### **Manifest Operations**
- **`manifest fetch`** вЂ” Fetch manifest content from a registry (supports all OCI media types)
- **`manifest push`** вЂ” Push manifest content to a registry from a file or stdin
- **`manifest delete`** вЂ” Delete a manifest from a registry by digest
- **`manifest fetch-config`** вЂ” Fetch the config blob of a manifest

### **Blob Operations**
- **`blob fetch`** вЂ” Fetch raw blob content from a registry by digest
- **`blob push`** вЂ” Push blob content to a registry from a file or stdin
- **`blob delete`** вЂ” Delete a blob from a registry by digest

### **Repository Operations**
- **`repo ls`** вЂ” List repositories in a registry (catalog API)
- **`repo tags`** вЂ” List all tags for a repository

### **Backup & Restore**
- **`backup`** вЂ” Backup artifacts to local OCI layout directory
- **`restore`** вЂ” Restore artifacts from local OCI layout to a registry

## рџ”ђ 3-Tier Authentication Waterfall

Introduced a robust **RegistryService** with intelligent credential resolution:

1. **Explicit credentials** вЂ” Use credentials provided via `--username`/`--password` flags
2. **Stored credentials** вЂ” Fall back to credentials from Docker config (`~/.docker/config.json`) via `DockerConfigStore`
3. **Anonymous access** вЂ” Fall back to unauthenticated requests for public registries

This waterfall ensures maximum compatibility: explicitly passed credentials take priority, stored credentials are used when available, and public registries work seamlessly without authentication.

## рџ“Љ Rich Output Models

Added **7 new AOT-compatible output models** for structured JSON output:

- **`PushResult`** вЂ” Digest, size, media type for pushed artifacts
- **`PullResult`** вЂ” Digest, files downloaded, total size
- **`AttachResult`** вЂ” Digest of the attached referrer artifact
- **`TagResult`** вЂ” Target digest and new tag name
- **`ListResult`** вЂ” Repository lists, tag lists with pagination support
- **`DiscoverResult`** вЂ” Referrer manifests with artifact types
- **`DeleteResult`** вЂ” Confirmation of deleted manifest/blob digest

All models use **source-generated JSON serialization** (`JsonSerializerContext`) for Native AOT compatibility and fast cold-start performance.
